IFTTT is a public web service that allows you to have creative control and automated interactions between various cloud-based services like Twitter, Instagram, Dropbox, and more. The interactions between the services are considered IFTTT because it means “If this, then that”.

What we did during this lab with IFTTT was we set up the raspberry pi’s with a light sensor to sense when the light was high and when the light was low. What we did with IFTTT was we set it up so that when the light reached a certain threshold it would log the sunset in your google calendar. This showed us how you can use sensors and do things over the internet with the results of those sensors and although it didn't show how powerful this can be, the things you can do with those sensors and the internet are almost infinite.

We only used the raspberry pi and the google calendar but the IFTTT service has lots more than just that. I was looking around it and I saw things like Dominoes and the Apple App store and I can only imagine creating something that orders Dominoes for you at a certain time of day or temperature. All in all I don’t think there are many limitations on what you can do online and with a raspberry pi even with the things IFTTT does not have because you can always code what you need yourself if you really wanted to.

I was able to use the Java that I was taught this year to accomplish this DDM using 2D arrays, ArrayLists, Classes, Files, and more. The DDM was to simulate a crossword puzzle using the console in Java under some requirements that were given. Some of the things that were asked for were lackluster so some of the code seems forced and I think it could look a lot better if I were to fully do this the way I envisioned it. I was able to meet most of the requirements given but due to lack of time, some of them aren’t able to be fulfilled.

The Java program contains five classes and a text file that is being read from in the File Handler. There is a Clue class that contains all of the information for each word provided in the text file. It is using the data grabbed from the File Handler in order to find this data and then provide it to the other classes. There is also the Square class which only contains the char inside of an array that is used to populate the char array used. Here this class seems useless but if the program were to be expanded upon it would be nice to be able to store more things inside of the class. The second to last class, the Main class contains the loops and User Interface to use all of the methods from the Crossword class.

The Crossword class is where most of the big methods are that control the bulk of the program. Here, I used three 2D arrays to figure out most of the calculations to check if the user had completed the crossword. Two of the arrays are char arrays, one being the user’s array and the other being a completely finished array. The finished array is filled out using the Square array, the third array, which got all of its information from the words provided in the text file. To check if the user is correct when submitting the crossword, it compares the two arrays and checks to ensure each index is the same. If it is, it congratulates the user and tells them that they won. Otherwise, it will tell the user they lost and end the program.

Looking back, I think I should have sent the user back into the loop if they messed up the crossword instead of ending the program and having them lose their progress which wouldn’t be a hard fix. I also think instead of coding the numbers into the first position of the word it could’ve been to the side to avoid confusion on what number each word was once they had filled in the word.. There is also the “problem” where if you erase a word that crosses another word it will delete the letter that is within the other word. This could be looked at as a feature as you are deleting the word, but I think I could fix this issue by storing each and every word that has been guessed and their indexes and keeping them true whilst they are not guessing that particular word number. I could then use that data to make sure those indexes aren’t changed during the erasing process. Overall this program isn’t my best work but it does what it is supposed to do and provides the experience that it was supposed to provide.

My Monopoly game was made in JFX where I use an SQL Database to store information about the board as well as save different games for players that can be loaded. I use scene switching as well as a Utils class to keep everything organized and keep static methods for the database in their own class. A Cells class keeps track of the different places on the board and their data, getting this information from a database. The BoardController class will handle all of the actions the player makes such as turns, button presses, purchases, etc, and send this information to the correct class to be updated. The Player class will keep IDs of all the players as well as their data such as money and owned properties. In the game the player has $2500, can move around the board, buy property, and gain money when other people land on their property. The UI on the right will keep track of the player's information(money and properties, etc). If I were to continue working on this in the future, I would add the functionality for Community Chest / Chance cards as well as properties to make the game more like the original. Working on this project has allowed me to learn about new things I can implement in other programs such as animation in JavaFX. I now have different methods in my GitHub that I can reuse/rely on when/if I ever plan on animating anything in JFX in the future. Some other things I learned about are the use of enums on JFX. I thought they only existed in Unity, but found ways to utilize them in my Monopoly program to keep track of different cell types. I also learned from Mr. Riley’s feedback that you can put a function inside of an enum, isn’t that interesting!
